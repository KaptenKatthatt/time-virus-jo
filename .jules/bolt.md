## 2025-05-18 - [Unnecessary DB Call Due to Overwritten Property in Socket Updates]
**Learning:** In `backend/src/controllers/socket.controller.ts`, `buildLobbyUpdate` queried `getAllPlayers()` from the database just to assign it to `onlinePlayers`. However, its wrapper `buildLobbyUpdateForIo` immediately overwrote `onlinePlayers` with a fresh list obtained from `io.fetchSockets()`. This resulted in a redundant database query being executed on *every* lobby update operation.
**Action:** Always verify if a returned database property is actually consumed by the caller, especially when working with wrapper functions that inject real-time or augmented state into the base payloads.

## 2026-05-24 - [Avoid Synchronous DB Validation in High-Frequency Socket Payloads]
**Learning:** `buildBaseLobbyUpdate` was synchronously querying the database with `findExistingGameIds` to validate whether each game in the `activeGames` memory object still existed. This caused an unnecessary N+1-like performance bottleneck because lobby updates are frequently broadcast to all clients (`updateLobbyForAll` is called on connect, disconnect, and during game events). Since the backend already employs an asynchronous background reconcile loop (`startReconcileCleanup`) and connection event handlers to keep `activeGames` in sync with the database, this synchronous check was redundant and impacted performance at scale.
**Action:** When a real-time system already has an async background reconcile task and connection lifecycle hooks managing an in-memory active state, trust the memory state for frequent broadcasts. Avoid injecting synchronous validation queries into high-frequency emission paths.
