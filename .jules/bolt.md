## 2025-05-18 - [Unnecessary DB Call Due to Overwritten Property in Socket Updates]
**Learning:** In `backend/src/controllers/socket.controller.ts`, `buildLobbyUpdate` queried `getAllPlayers()` from the database just to assign it to `onlinePlayers`. However, its wrapper `buildLobbyUpdateForIo` immediately overwrote `onlinePlayers` with a fresh list obtained from `io.fetchSockets()`. This resulted in a redundant database query being executed on *every* lobby update operation.
**Action:** Always verify if a returned database property is actually consumed by the caller, especially when working with wrapper functions that inject real-time or augmented state into the base payloads.
## 2024-05-23 - Remove synchronous database validation on lobby update broadcast
**Learning:** The application maintains live games in memory (`activeGames`) and asynchronously cleans up stale states using connection/disconnection handlers and a recurring `startReconcileCleanup` process. Synchronously checking the database to validate the existence of these games upon every lobby broadcast (`buildBaseLobbyUpdate`) creates severe bottlenecks since `lobby:update` events happen frequently (chat, new connection, scoring). Database state checking was redundant and caused N+1 issues when broadcasting heavily to all clients.
**Action:** Remove `findExistingGameIds` call in the lobby broadcast path. Always trust the synchronously updated memory state which is naturally kept in-sync by `startReconcileCleanup`, thereby saving an expensive, un-cached network/db roundtrip per broadcast payload generation.
